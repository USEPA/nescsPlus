Which search is best for me?
The NESCS Plus Webtool offers two ways for users to identify and develop lists of Final Ecosystem Services (or FES components) that are of interest to them.
  1. Browse Core Options allows the user to browse, filter, and select from a dataset containing ~1,000 pre-specified FES options. Each FES option is comprised of one element (class or subclass) from each classification component: Environment, EEP, Direct Use/Non-Use, Direct User, and Beneficiary components. The Core Options have been pre-selected by the NESCS Plus developers to represent the FES options that are likely to be most common or plausible (based on subjective judgment).
  2. Query All Options allows the user to consider and select from all possible FES options (>75,000 options). To define each distinct FES of interest, the user must select one element (class or subclass) from each classification component.
Which One is Better for Me? Using the Core Options may be more appropriate for users who have less experience using NESCS Plus or are less certain about the exact FES options they want to select. It makes it easier to do exploratory or preliminary selections of FES, but it also relies on judgments made by the NESCS Plus developers regarding the FES options that are most common or plausible. Using All Options offers the user more flexibility but places more burden on the user to select the FES options that are most relevant for his or her needs.
